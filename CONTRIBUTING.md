# Contributing to AnthraCode

Thanks for your interest in AnthraCode. This repository is the **public home** of
the project — it hosts the documentation, releases information, and issue
tracker. The application's source code is maintained privately; the product is
distributed as a compiled binary via npm (`anthracode-ai`).

There are several valuable ways to contribute.

## Report a bug

Open an issue and include:

- AnthraCode version (`anthracode --version`),
- your OS and architecture (Windows x64 / Linux x64),
- exact steps to reproduce,
- what you expected vs what happened,
- relevant logs or error output (redact secrets and file contents you don't want
  public).

For **security vulnerabilities**, do **not** open a public issue — follow
`SECURITY.md`.

## Request a feature

Open an issue describing the problem you're trying to solve (not only the
solution you have in mind). Real use cases help us prioritize.

## Improve the docs

The README and the documents in this repository can be improved by anyone. Open
a pull request against this **public** repository for:

- README clarity, install instructions, examples,
- typo and accuracy fixes,
- screenshots and usage guides.

> Do not submit source code to this repository — it is docs-only by design (see
> the repository `.gitignore`). Source-code contributions are not accepted here.

## Ground rules

- Be respectful — see `CODE_OF_CONDUCT.md`.
- By contributing to this repository, you agree your contributions are provided
  under the repository's MIT License.
- Keep discussions focused and in good faith.

## Questions

Open a discussion or issue, or email **hello@anthracode.com**.
